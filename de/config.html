gallery-dl Konfigurationsdatei unter Windows — Einrichtungsanleitung

Die gallery-dl Konfigurationsdatei ermöglicht dauerhafte Standardwerte — Ausgabeordner, Dateinamen-Vorlagen, Rate-Limiting, Cookies und seitenspezifische Optionen. Kein Wiederholen von Flags bei jedem Befehl.

Speicherort der Konfigurationsdatei

Wo sucht gallery-dl unter Windows nach der Konfigurationsdatei?

gallery-dl prüft diese Speicherorte der Reihe nach — erster Treffer gewinnt.

Priorität 1
%APPDATA%\gallery-dl\config.json

z. B. C:\Users\IhrName\AppData\Roaming\gallery-dl\config.json — empfohlen für dauerhafte Einrichtung

Priorität 2
gallery-dl.conf (gleicher Ordner wie gallery-dl.exe)

Portable Einrichtung — Konfiguration reist mit der ausführbaren Datei

CLI-Flag
gallery-dl --config "C:\Pfad\zu\meinekonfig.json" "URL"

Konfigurationspfad pro Befehl überschreiben

Tipp: Ordner %APPDATA%\gallery-dl\ erstellen falls nicht vorhanden. Win+R drücken, %APPDATA% eingeben, Enter drücken, dann neuen Ordner gallery-dl erstellen.

Starter-Konfiguration

Empfohlene gallery-dl-Konfiguration für Windows — kopieren & einfügen

Als config.json in %APPDATA%\gallery-dl\ speichern. Pfade und Optionen nach Bedarf anpassen.

gallery-dl.conf — empfohlener Windows-Starter
{ "extractor": { "base-directory": "C:/Bilder/", "filename": "{category}_{author}_{num:>04}_{filename}.{extension}", "sleep-request": 1.5, "retries": 5, "timeout": 30, "verify": true }, "downloader": { "part": true, "rate": "2M" }, "output": { "mode": "terminal", "progress": true, "shorten": true } }
Hinweis: JSON erfordert doppelte Anführungszeichen und keine nachgestellten Kommas. Vorwärtsschrägstriche / in Pfaden verwenden — funktionieren unter Windows einwandfrei und vermeiden JSON-Escape-Probleme mit Backslashes.

Wichtige Optionen erklärt

Nützlichste gallery-dl-Konfigurationsoptionen für Windows

Download-Ordner festlegen — gilt für alle Seiten
"base-directory": "C:/Bilder/"
Benutzerdefinierte Dateinamen-Vorlage — Kategorie + Autor + Nummer + Originalname
"filename": "{category}_{author}_{num:>04}_{filename}.{extension}"
Seitenspezifischer Ausgabeordner — Instagram in eigenen Unterordner
"directory": ["{category}", "{uploader}"]
Anfragen verlangsamen um Rate-Limiting zu vermeiden (Sekunden zwischen Anfragen)
"sleep-request": 2.0
Download-Geschwindigkeit begrenzen (nützlich bei geteilten Verbindungen)
"rate": "2M"
Cookies-Datei für bestimmte Seite setzen (Instagram-Beispiel)
"instagram": { "cookies": "C:/Tools/gallery-dl/instagram-cookies.txt" }

FAQ

gallery-dl Konfiguration — häufige Fragen

Wo befindet sich die gallery-dl Konfigurationsdatei unter Windows?
Der Standardspeicherort ist %APPDATA%\gallery-dl\config.json. Ausführen (Win+R), %APPDATA%\gallery-dl eingeben und Enter drücken, um direkt dorthin zu navigieren. Alternativ gallery-dl.conf im gleichen Ordner wie gallery-dl.exe für portable Einrichtung ablegen.
gallery-dl meldet "Konfigurationsdatei nicht gefunden" — wie beheben?
gallery-dl benötigt keine Konfigurationsdatei — funktioniert auch ohne. Für eine eigene: Ordner %APPDATA%\gallery-dl\ erstellen und JSON-Datei als config.json speichern. Gültiges JSON sicherstellen — ein fehlendes Komma oder Anführungszeichen bricht alles.
Wie validiere ich meine gallery-dl JSON-Konfiguration?
Konfiguration bei jsonlint.com einfügen, um auf Syntaxfehler zu prüfen. Häufige Fehler: nachgestellte Kommas nach dem letzten Element, einzelne statt doppelte Anführungszeichen, Windows-Backslashes in Pfaden (Vorwärtsschrägstriche oder \\ verwenden).
Kann ich verschiedene Einstellungen für verschiedene Seiten haben?
Ja. Seitenspezifische Einstellungen unter dem Seiten-Schlüssel einbetten. Beispiel: "instagram": {"sleep-request": 3} innerhalb des "extractor"-Objekts. Seiten-Schlüssel entsprechen der {category}-Variable — instagram, twitter, pixiv, deviantart, reddit usw.
