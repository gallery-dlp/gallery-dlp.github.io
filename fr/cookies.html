gallery-dl Cookies & Connexion sur Windows — Guide d'Authentification

De nombreux sites nécessitent une connexion pour télécharger des galeries privées. gallery-dl lit les cookies directement depuis votre navigateur — sans stocker de mots de passe, sans exportation manuelle pour Chrome, Firefox et Edge.

⬇ Télécharger gallery-dl.exe Guide de configuration →

Méthode la plus rapide

Passer les cookies du navigateur à gallery-dl en une commande

Fonctionne pour la plupart des sites — Instagram, Twitter/X, Pixiv, DeviantArt, Reddit, Patreon et plus. Assurez-vous d'être connecté au site dans votre navigateur d'abord.

Utiliser les cookies de Chrome (le plus courant)
gallery-dl --cookies-from-browser chrome "URL"
Utiliser les cookies de Firefox
gallery-dl --cookies-from-browser firefox "URL"
Utiliser les cookies de Microsoft Edge
gallery-dl --cookies-from-browser edge "URL"
Utiliser les cookies de Brave
gallery-dl --cookies-from-browser brave "URL"
Important : Le navigateur doit être complètement fermé lorsque gallery-dl lit les cookies sur Windows. Si vous obtenez une erreur « database locked », fermez complètement Chrome/Firefox — y compris dans la barre des tâches — puis réessayez.

Méthode fichier cookies.txt

Utiliser un fichier cookies.txt avec gallery-dl sur Windows

Plus fiable pour l'automatisation et les tâches planifiées — exportez une fois, réutilisez plusieurs fois.

Étape 01

Installer une extension d'export de cookies

Installez Get cookies.txt LOCALLY depuis le Chrome Web Store ou Firefox Add-ons. Exporte au format Netscape compris par gallery-dl.

Étape 02

Se connecter au site cible

Ouvrez le site dans votre navigateur et assurez-vous d'être entièrement connecté. Complétez le 2FA si nécessaire.

Étape 03

Exporter les cookies pour ce site

Cliquez sur l'icône de l'extension sur le site → Exporter. Sauvegardez comme cookies.txt dans votre dossier gallery-dl.

Étape 04

Passer le fichier à gallery-dl

Utilisez le flag --cookies ou ajoutez le chemin dans le fichier de configuration pour une utilisation permanente.

Utiliser un fichier cookies.txt — CLI
gallery-dl --cookies "C:\Tools\gallery-dl\instagram-cookies.txt" "URL"
Définir le fichier de cookies de façon permanente dans config.json (par site)
"instagram": { "cookies": "C:/Tools/gallery-dl/instagram-cookies.txt" }

Authentification par site

Conseils d'authentification spécifiques par site

Instagram

Requis pour les comptes privés et les téléchargements en haute résolution. Connectez-vous via le navigateur, puis :

gallery-dl --cookies-from-browser chrome "https://www.instagram.com/username/"

Twitter / X

Requis pour les comptes privés. Les cookies Twitter expirent fréquemment — ré-exportez si ça ne fonctionne plus.

gallery-dl --cookies-from-browser chrome "https://x.com/username/media"

Pixiv

Requis pour le contenu R-18 et les téléchargements complets. Connectez-vous sur pixiv.net et passez les cookies.

gallery-dl --cookies-from-browser chrome "https://www.pixiv.net/users/USER_ID"

Patreon

Requis pour le contenu réservé aux mécènes. Vous devez avoir un abonnement actif au créateur.

gallery-dl --cookies-from-browser chrome "https://www.patreon.com/creator"

FAQ

gallery-dl cookies — questions fréquentes

gallery-dl dit « Connexion requise » même avec des cookies
Vos cookies ont peut-être expiré. Déconnectez-vous et reconnectez-vous au site, puis réexécutez avec --cookies-from-browser chrome. Assurez-vous également que le navigateur est complètement fermé avant que gallery-dl lise sa base de données de cookies sur Windows.
« database is locked » lors de l'utilisation de --cookies-from-browser sur Windows
Chrome et Edge verrouillent leur base de données SQLite de cookies pendant leur exécution. Fermez complètement toutes les fenêtres du navigateur (vérifiez aussi la barre des tâches), puis réexécutez gallery-dl. Alternativement, exportez vers un fichier cookies.txt et utilisez --cookies.
Est-il sûr de passer des cookies du navigateur à gallery-dl ?
gallery-dl lit les cookies localement depuis votre navigateur — ils ne quittent jamais votre machine. L'outil est open source et audité. Ne partagez jamais votre fichier cookies.txt — il accorde un accès complet au compte. Supprimez le fichier quand vous n'en avez plus besoin.
gallery-dl supporte-t-il la connexion avec nom d'utilisateur et mot de passe ?
Certains extracteurs supportent la connexion directe avec -u USERNAME -p PASSWORD. Cependant, cela ne fonctionne pas pour les sites avec 2FA ou les flux d'authentification modernes. La méthode --cookies-from-browser est plus fiable et recommandée pour la plupart des sites.
Comment utiliser les cookies dans le fichier de configuration gallery-dl ?
Ajoutez le chemin des cookies sous la clé du site dans config.json : "extractor": { "instagram": { "cookies": "C:/Tools/gallery-dl/cookies.txt" } } Cela applique automatiquement les cookies pour chaque téléchargement Instagram sans avoir besoin du flag CLI.
