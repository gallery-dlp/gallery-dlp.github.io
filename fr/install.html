Installation étape par étape

Installer gallery-dl sur Windows en 5 étapes

Fonctionne sur Windows 11 et Windows 10 (64 bits). Sans assistant d'installation, sans droits administrateur.

Étape 01

Télécharger gallery-dl.exe

Allez sur la page de téléchargement et sauvegardez gallery-dl.exe depuis la version officielle GitHub.

Étape 02

Sauvegarder dans un dossier dédié

Déplacez le fichier dans un emplacement permanent, ex : C:\Tools\gallery-dl\. Évitez Téléchargements — les fichiers peuvent être supprimés automatiquement.

Étape 03

Ouvrir PowerShell dans ce dossier

Dans l'Explorateur, cliquez sur la barre d'adresse, tapez powershell et appuyez sur Entrée. PowerShell s'ouvre directement dans le bon dossier.

Étape 04

Vérifier que ça fonctionne

Exécutez gallery-dl --version. Un numéro de version comme 1.28.x devrait apparaître. Si oui, ça fonctionne.

Étape 05

Mettre à jour vers la dernière version

Exécutez gallery-dl -U. Télécharge et remplace automatiquement par la dernière version. Faites-le régulièrement.

Optionnel

Configurer un fichier de configuration

Créez un gallery-dl.conf pour définir les dossiers de sortie par défaut, les modèles de noms de fichiers et les cookies. Guide de configuration →

Optionnel mais recommandé

Comment ajouter gallery-dl au PATH sur Windows

Avec gallery-dl dans le PATH, vous pouvez exécuter gallery-dl depuis n'importe quel dossier dans PowerShell ou CMD.

Méthode graphique (recommandée)

  1. Appuyez sur Win et recherchez Variables d'environnement
  2. Cliquez sur Modifier les variables d'environnement système
  3. Dans Variables utilisateur, sélectionnez Path → Modifier
  4. Cliquez sur Nouveau et collez le chemin du dossier, ex : C:\Tools\gallery-dl
  5. Cliquez sur OK sur tous les dialogues et redémarrez le terminal

Méthode PowerShell

Exécutez une fois dans PowerShell — définit le PATH uniquement pour votre utilisateur, sans droits admin :

setx PATH "%PATH%;C:\Tools\gallery-dl"

Fermez et rouvrez PowerShell. Testez avec gallery-dl --version depuis n'importe quel répertoire.

Note : Si vous voyez « gallery-dl n'est pas reconnu » après l'ajout au PATH, fermez toutes les fenêtres du terminal et rouvrez-les — les changements de PATH ne prennent effet que dans les nouvelles sessions.

Problèmes courants

Résoudre les problèmes d'installation de gallery-dl sur Windows

Avertissement Windows SmartScreen lors de l'exécution de gallery-dl.exe
Cliquez sur Informations complémentaires → Exécuter quand même. SmartScreen signale les nouveaux exécutables par défaut. gallery-dl.exe est la version officielle open source non modifiée de GitHub. Alternativement : clic droit → Propriétés → cochez Débloquer en bas → OK.
« gallery-dl n'est pas reconnu » dans PowerShell
Deux options : exécutez gallery-dl directement depuis son dossier, ou ajoutez le dossier au PATH (voir section ci-dessus). Après l'ajout au PATH, fermez toutes les fenêtres du terminal et rouvrez-les.
« Accès refusé » ou fichier bloqué par l'antivirus
Clic droit sur gallery-dl.exe → Propriétés → cochez Débloquer en bas si visible → OK. Si l'antivirus bloque, ajoutez une exception — c'est un faux positif pour les portables open source.
gallery-dl ne télécharge rien / se ferme immédiatement
Le site nécessite probablement une connexion. Exécutez gallery-dl --cookies-from-browser chrome "URL". Voir le guide des cookies pour les détails complets.
