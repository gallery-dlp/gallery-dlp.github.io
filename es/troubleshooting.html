gallery-dl No Funciona en Windows — Corregir Errores Comunes

La mayoría de errores de gallery-dl en Windows tienen soluciones sencillas. Empieza aquí: actualiza con gallery-dl -U y comprueba las cookies. Esto resuelve el 80% de los problemas.

Empieza siempre aquí:

Paso 1 — actualizar gallery-dl
gallery-dl -U
Paso 2 — salida detallada para ver el error completo
gallery-dl -v "URL"

Referencia de errores

Mensajes de error de gallery-dl y cómo corregirlos

«Login required» o «Authentication required»
Causa: El sitio requiere que hayas iniciado sesión para acceder a este contenido.

Solución: Inicia sesión en el sitio en Chrome (o Firefox/Edge), cierra completamente el navegador, luego ejecuta: gallery-dl --cookies-from-browser chrome "URL" Si las cookies siguen expirando, exporta un archivo cookies.txt. Ver la guía de cookies.
«gallery-dl no se reconoce como comando»
Causa: PowerShell no encuentra gallery-dl.exe porque su carpeta no está en el PATH.

Solución — opción 1: Navega hasta la carpeta que contiene gallery-dl.exe en el Explorador, haz clic en la barra de direcciones, escribe powershell, pulsa Intro. Ejecuta gallery-dl desde allí.

Solución — opción 2: Añade la carpeta de gallery-dl al PATH. Ver la guía de instalación.
Error HTTP 403 Forbidden
Causa: El sitio está bloqueando la petición — rate limiting, inicio de sesión requerido o bloqueo geográfico.

Solución: Primero actualiza: gallery-dl -U. Luego intenta con cookies: gallery-dl --cookies-from-browser chrome "URL" Si usas VPN, prueba sin ella. Añade --sleep-request 3 si estás alcanzando los límites de velocidad.
gallery-dl no descarga nada — se cierra inmediatamente
Causa: Inicio de sesión requerido, formato de URL no admitido o extractor necesita actualización.

Solución: Ejecuta con salida detallada: gallery-dl -v "URL" Luego intenta con cookies: gallery-dl --cookies-from-browser chrome "URL". Actualiza: gallery-dl -U.
Error de certificado SSL en Windows
Causa: El almacén de certificados de Windows está desactualizado, o un proxy/VPN intercepta HTTPS.

Solución: Ejecuta Windows Update (Configuración → Windows Update), luego vuelve a intentarlo. Como último recurso: gallery-dl --no-check-certificate "URL" — no recomendado a largo plazo.
Windows SmartScreen bloquea gallery-dl.exe
Causa: SmartScreen señala los ejecutables portátiles no firmados por defecto — esto es un falso positivo.

Solución: Haz clic en Más información → Ejecutar de todos modos. O clic derecho en gallery-dl.exe → Propiedades → marca Desbloquear en la parte inferior → Aceptar.
«database is locked» al usar --cookies-from-browser
Causa: Chrome o Edge bloquea su base de datos SQLite de cookies mientras está en ejecución.

Solución: Cierra completamente el navegador — comprueba también el icono en la bandeja del sistema. Luego vuelve a ejecutar gallery-dl. Alternativamente, exporta cookies a un archivo cookies.txt y usa --cookies.
gallery-dl está muy lento — las descargas se bloquean o se detienen
Causa: Rate limiting del sitio, conexión inestable o VPN throttling.

Solución: Añade --sleep-request 3 --retries 5 a tu comando. Para bloqueos persistentes, prueba --part para reanudar descargas parciales.
