gallery-dl Cookies & Login no Windows — Guia de Autenticação

Muitos sites requerem login para baixar galerias privadas. O gallery-dl lê cookies diretamente do seu navegador — sem senhas armazenadas, sem exportação manual para Chrome, Firefox e Edge.

⬇ Baixar gallery-dl.exe Guia de configuração →

Método mais rápido

Passe cookies do navegador para gallery-dl em um comando

Funciona para a maioria dos sites — Instagram, Twitter/X, Pixiv, DeviantArt, Reddit, Patreon e mais. Certifique-se de estar logado no site no seu navegador primeiro.

Usar cookies do Chrome (mais comum)
gallery-dl --cookies-from-browser chrome "URL"
Usar cookies do Firefox
gallery-dl --cookies-from-browser firefox "URL"
Usar cookies do Microsoft Edge
gallery-dl --cookies-from-browser edge "URL"
Usar cookies do Brave
gallery-dl --cookies-from-browser brave "URL"
Importante: O navegador deve estar completamente fechado quando o gallery-dl lê os cookies no Windows. Se aparecer erro "database locked", feche completamente o Chrome/Firefox — inclusive na bandeja do sistema — e tente novamente.

Método arquivo cookies.txt

Usando arquivo cookies.txt com gallery-dl no Windows

Mais confiável para automação e tarefas agendadas — exporte uma vez, reutilize várias vezes.

Passo 01

Instalar extensão de exportação de cookies

Instale Get cookies.txt LOCALLY da Chrome Web Store ou Firefox Add-ons. Exporta no formato Netscape que gallery-dl entende.

Passo 02

Fazer login no site de destino

Abra o site no navegador e certifique-se de estar totalmente logado. Complete qualquer 2FA se necessário.

Passo 03

Exportar cookies para aquele site

Clique no ícone da extensão no site → Exportar. Salve como cookies.txt na pasta do gallery-dl.

Passo 04

Passar o arquivo para gallery-dl

Use a flag --cookies ou adicione o caminho ao arquivo de configuração para uso permanente.

Usar arquivo cookies.txt — CLI
gallery-dl --cookies "C:\Tools\gallery-dl\instagram-cookies.txt" "URL"
Definir arquivo de cookies permanentemente no config.json (por site)
"instagram": { "cookies": "C:/Tools/gallery-dl/instagram-cookies.txt" }

Autenticação por site

Dicas de autenticação específicas por site

Instagram

Necessário para contas privadas e downloads em alta resolução. Faça login pelo navegador, depois:

gallery-dl --cookies-from-browser chrome "https://www.instagram.com/username/"

Twitter / X

Necessário para contas privadas. Os cookies do Twitter expiram com frequência — re-exporte se parar de funcionar.

gallery-dl --cookies-from-browser chrome "https://x.com/username/media"

Pixiv

Necessário para conteúdo R-18 e downloads completos. Faça login em pixiv.net e passe os cookies.

gallery-dl --cookies-from-browser chrome "https://www.pixiv.net/users/USER_ID"

Patreon

Necessário para conteúdo exclusivo de patrocinadores. Você deve ter uma assinatura ativa do criador.

gallery-dl --cookies-from-browser chrome "https://www.patreon.com/creator"

Perguntas Frequentes

gallery-dl cookies — perguntas frequentes

gallery-dl diz "Login necessário" mesmo com cookies
Seus cookies podem ter expirado. Faça logout e login novamente no site, depois execute novamente com --cookies-from-browser chrome. Para arquivos cookies.txt, exporte uma cópia nova. Certifique-se também de que o navegador está completamente fechado antes do gallery-dl ler seu banco de dados de cookies no Windows.
"database is locked" ao usar --cookies-from-browser no Windows
Chrome e Edge bloqueiam seu banco de dados SQLite de cookies enquanto estão em execução. Feche todas as janelas do navegador completamente (verifique também a bandeja do sistema), depois execute o gallery-dl novamente. Alternativamente, exporte para um arquivo cookies.txt e use --cookies.
É seguro passar cookies do navegador para gallery-dl?
O gallery-dl lê cookies localmente do seu navegador — eles nunca saem do seu computador. A ferramenta é código aberto e auditada. Nunca compartilhe seu arquivo cookies.txt — ele concede acesso completo à conta. Delete o arquivo quando não precisar mais.
gallery-dl suporta login com usuário e senha?
Alguns extratores suportam login direto com -u USERNAME -p PASSWORD. Porém isso não funciona para sites com 2FA ou fluxos de autenticação modernos. O método --cookies-from-browser é mais confiável e recomendado para a maioria dos sites.
Como uso cookies no arquivo de configuração do gallery-dl?
Adicione o caminho dos cookies sob a chave do site em config.json: "extractor": { "instagram": { "cookies": "C:/Tools/gallery-dl/cookies.txt" } } Isso aplica cookies automaticamente para cada download do Instagram sem precisar da flag CLI.
