gallery-dl Não Funciona no Windows — Corrigir Erros Comuns

A maioria dos erros do gallery-dl no Windows tem soluções simples. Comece aqui: atualize com gallery-dl -U e verifique os cookies. Isso resolve 80% dos problemas.

Sempre comece aqui:

Passo 1 — atualizar gallery-dl
gallery-dl -U
Passo 2 — saída detalhada para ver o erro completo
gallery-dl -v "URL"

Referência de erros

Mensagens de erro do gallery-dl e como corrigi-las

"Login required" ou "Authentication required"
Causa: O site requer que você esteja logado para acessar este conteúdo.

Solução: Faça login no site no Chrome (ou Firefox/Edge), feche o navegador completamente, depois execute: gallery-dl --cookies-from-browser chrome "URL" Se os cookies continuarem expirando, exporte um arquivo cookies.txt. Veja o guia de cookies.
"gallery-dl não é reconhecido como comando"
Causa: PowerShell não encontra gallery-dl.exe porque sua pasta não está no PATH.

Solução — opção 1: Navegue até a pasta contendo gallery-dl.exe no Explorer, clique na barra de endereços, digite powershell, pressione Enter. Execute gallery-dl de lá.

Solução — opção 2: Adicione a pasta do gallery-dl ao PATH. Veja o guia de instalação.
HTTP erro 403 Forbidden
Causa: O site está bloqueando a requisição — rate limiting, login necessário ou bloqueio geográfico.

Solução: Primeiro atualize: gallery-dl -U. Depois tente com cookies: gallery-dl --cookies-from-browser chrome "URL" Se estiver usando VPN, tente desativá-la. Adicione --sleep-request 3 se estiver atingindo rate limits.
gallery-dl não baixa nada — fecha imediatamente
Causa: Login necessário, formato de URL não suportado ou extrator precisa de atualização.

Solução: Execute com saída detalhada: gallery-dl -v "URL" Depois tente com cookies: gallery-dl --cookies-from-browser chrome "URL". Atualize: gallery-dl -U.
Erro de certificado SSL no Windows
Causa: Repositório de certificados do Windows desatualizado, ou proxy/VPN interceptando HTTPS.

Solução: Execute Windows Update (Configurações → Windows Update), depois tente novamente. Como último recurso apenas: gallery-dl --no-check-certificate "URL" — não recomendado a longo prazo.
Windows SmartScreen bloqueia gallery-dl.exe
Causa: SmartScreen sinaliza executáveis portáteis não assinados por padrão — isso é um falso positivo.

Solução: Clique em Mais informações → Executar mesmo assim. Ou clique com o botão direito em gallery-dl.exe → Propriedades → marque Desbloquear na parte inferior → OK.
"database is locked" ao usar --cookies-from-browser
Causa: Chrome ou Edge bloqueia seu banco de dados SQLite de cookies enquanto está em execução.

Solução: Feche o navegador completamente — incluindo o ícone na bandeja do sistema. Depois execute gallery-dl novamente. Alternativamente, exporte cookies para um arquivo cookies.txt e use --cookies.
gallery-dl está muito lento — downloads travam ou param
Causa: Rate limiting pelo site, conexão instável ou VPN throttling.

Solução: Adicione --sleep-request 3 --retries 5 ao comando. Para travamentos persistentes, tente --part para retomar downloads parciais.
